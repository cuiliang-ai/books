## 顶层模块一览

`src-tauri/src/` 的模块声明（来自 `lib.rs` 顶部）：

```rust
mod app_config;        // AppType / MultiAppConfig / McpServer / SkillApps
mod app_store;         // Tauri Store 封装
mod auto_launch;       // 开机自启 (auto-launch crate 封装)
mod claude_mcp;        // Claude 的 MCP 文件操作
mod claude_plugin;     // Claude 自定义插件
mod codex_config;      // Codex config.toml + auth.json 原子写入
mod commands;          // 所有 IPC 命令（30 子模块）
mod config;            // 文件 IO 基础（atomic_write / home_dir）
mod database;          // SQLite DAO
mod deeplink;          // ccswitch:// 协议
mod error;             // AppError 统一错误类型
mod gemini_config;     // Gemini settings.json
mod gemini_mcp;        // Gemini MCP 操作
mod init_status;       // 初始化错误状态
mod lightweight;       // 轻量模式（最小资源占用）
mod linux_fix;         // Linux 特定兼容
mod mcp;               // MCP 解析 + 工具
mod openclaw_config;   // OpenClaw 配置
mod opencode_config;   // OpenCode 配置
mod panic_hook;        // panic 捕获
mod prompt;            // Prompt 最小封装
mod prompt_files;      // Prompt 文件
mod provider;          // Provider 核心模型
mod provider_defaults; // 50+ Provider 预设
mod proxy;             // 本地 HTTP 代理子系统
mod services;          // 业务服务层
mod session_manager;   // 跨工具会话
mod settings;          // AppSettings（用户偏好）
mod store;             // AppState（运行时全局状态）
mod tray;              // 系统托盘
mod usage_script;      // 用量查询 JS 脚本
```

## 按职责分组

### Group 1：应用基础设施

| 模块 | 职责 |
|------|------|
| `lib.rs` | 应用入口 / setup / RunEvent |
| `error.rs` | `AppError` + `thiserror` |
| `panic_hook.rs` | panic → 日志 + 对话框 |
| `init_status.rs` | 启动失败状态持久化 |
| `settings.rs` | 用户偏好配置（742 行） |
| `store.rs` | AppState 薄封装 |
| `app_store.rs` | Tauri Store 插件封装 |
| `lightweight.rs` | 最小资源模式 |
| `linux_fix.rs` | WebKit2GTK 兼容 |

### Group 2：领域模型

| 模块 | 职责 | 行数 |
|------|------|------|
| `app_config.rs` | AppType / MultiAppConfig | ~700 |
| `provider.rs` | Provider / ProviderManager | 997 |
| `provider_defaults.rs` | 50+ 预设 | 238 |
