## 为什么值得读这本书

cc-switch 把很多**实际但不显眼**的工程难题解决得很漂亮：

- **原子写入** — 所有 live 配置使用 `temp file + rename` 模式，任何崩溃都不会破坏用户的 `~/.claude/settings.json`
- **本地 HTTP 代理** — 用 Rust 手写 hyper accept loop，保留原始 header casing，使上游请求在协议层面与 CLI 直连一致
- **熔断器 + Failover** — Closed / Open / HalfOpen 三态机，跨请求共享状态，支持错误率阈值 + 最小请求数双维度
- **思考预算矫正** — 针对 Claude extended thinking 的多层 rectifier，解决不同 Provider 的参数差异
- **多 App SSOT** — MCP 服务器 / Skills 在多个 CLI 间双向同步，单一数据源配合 `enabled_apps` 标记
- **跨进程会话检索** — Session Manager 解析 6 种 CLI 的会话格式（JSONL / SQLite 等），统一展示

本书把这些细节按"鸟瞰 → 骨架 → 数据 → 代理 → 扩展 → 存储 → 打包"的顺序拆开，帮助你：
1. **学习 Tauri 2 实战** — 不是 hello-world，而是完整的生产级应用
2. **学习 Rust 异步与网络编程** — Axum、Hyper、Tokio、rustls 的工业级组合
3. **学习 React 18 + shadcn/ui 组织大型前端** — Radix UI + TanStack Query + i18next + Zod
