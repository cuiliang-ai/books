## 痛点

每个 AI CLI 都有自己的会话存储：

| CLI | 格式 | 位置 |
|-----|------|-----|
| Claude Code | JSONL | `~/.claude/projects/*/sessions/*.jsonl` |
| Codex | JSONL + SQLite | `~/.codex/sessions/` |
| Gemini CLI | JSON | `~/.gemini/sessions/` |
| OpenCode | JSON | `~/.config/opencode/sessions/` |
| OpenClaw | SQLite | `~/.openclaw/db/` |

用户想"找到上周用 Claude 写 React 组件的那次对话" — 难。cc-switch 的 Session Manager 统一解析这些格式，提供一个跨 CLI 的会话浏览器。

## 统一数据模型

```rust
pub struct SessionMeta {
    pub id: String,
    pub app: AppType,
    pub project: Option<String>,
    pub title: Option<String>,
    pub started_at: DateTime<Utc>,
    pub ended_at: Option<DateTime<Utc>>,
    pub message_count: u32,
    pub model: Option<String>,
    pub total_tokens: Option<u64>,
    pub cost: Option<f64>,
}

pub struct SessionMessage {
    pub role: Role,                    // User / Assistant / Tool / System
    pub content: Vec<ContentBlock>,
    pub timestamp: DateTime<Utc>,
    pub tool_calls: Vec<ToolCall>,
}
```

不管底层是 JSONL 还是 SQLite，前端看到的都是这个模型。

## Claude Code: JSONL 解析

```rust
pub async fn list_claude_sessions() -> Result<Vec<SessionMeta>> {
    let project_dir = get_claude_projects_dir();
    let mut sessions = vec![];

    for project in fs::read_dir(&project_dir)? {
        let session_dir = project?.path().join("sessions");
        for entry in fs::read_dir(&session_dir)? {
            let path = entry?.path();
            // 懒解析：只读第一行（session start）+ 最后一行（session end）
            let meta = parse_session_meta_lazy(&path).await?;
            sessions.push(meta);
        }
    }
    Ok(sessions)
}
```

**懒解析**关键：列表视图不读整个文件，只读头尾——JSONL 大可达 MB 级。

## Full Load

点开单个会话时才完整解析：

```rust
pub async fn load_claude_session(id: &str) -> Result<SessionContent> {
    let path = find_session_path(id)?;
    let file = tokio::fs::File::open(&path).await?;
    let mut reader = BufReader::new(file).lines();
    let mut messages = vec![];

    while let Some(line) = reader.next_line().await? {
        let entry: Value = serde_json::from_str(&line)?;
        if let Some(msg) = parse_message(&entry) {
            messages.push(msg);
        }
    }

    Ok(SessionContent { messages, ... })
}
```

## OpenCode: 最复杂（997 行）

OpenCode 的 session 格式最复杂：
- 多文件（每个 session 分 manifest + parts）
- 工具调用嵌套
- 多模态内容 base64 存储

这是 `opencode.rs` 接近 1K 行的原因。
