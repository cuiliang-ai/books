## 托盘的价值

对"切换 Provider"这种高频操作，打开应用窗口太重。托盘菜单让用户：

- 右键托盘图标 → 看到当前 App 的 Provider 列表
- 点击目标 Provider → 立刻切换
- 托盘图标实时反映代理状态（绿点 = 运行中）

## tray.rs 结构

`src-tauri/src/tray.rs` (481 行):

```rust
pub fn init_tray(app: &App) -> Result<(), AppError> {
    let menu = build_tray_menu(app.state::<AppState>(), &get_db(app))?;
    let icon = load_tray_icon(&app.handle())?;

    TrayIconBuilder::new()
        .tooltip("CC Switch")
        .icon(icon)
        .menu(&menu)
        .on_menu_event(|app, event| {
            handle_menu_click(app, event.id());
        })
        .on_tray_icon_event(|tray, event| {
            if let TrayIconEvent::Click { button: MouseButton::Left, .. } = event {
                show_main_window(tray.app_handle());
            }
        })
        .build(app)?;

    Ok(())
}
```

### 菜单更新时机

- Provider 被添加 / 删除 / 重命名
- Provider 切换（高亮变化）
- Proxy / Failover 开关变化
- Language 切换（菜单标签 i18n）

每次都**重建菜单**：

```rust
pub fn update_tray_menu(app: &AppHandle, db: &Database) -> Result<(), AppError> {
    let tray = app.tray_by_id("main").ok_or(...)?;
    let menu = build_tray_menu(app.state::<AppState>(), db)?;
    tray.set_menu(Some(menu))?;
    Ok(())
}
```

## 图标

### 多分辨率 + 主题

macOS 暗色模式需要白色图标，亮色需要黑色：

```rust
#[cfg(target_os = "macos")]
fn select_tray_icon_for_appearance() -> Image<'_> {
    if is_dark_mode() {
        Image::from_bytes(include_bytes!("../icons/tray-white.png"))
    } else {
        Image::from_bytes(include_bytes!("../icons/tray-black.png"))
    }
}
```

Windows / Linux 用彩色图标，无需区分。

## 跨平台细节

### macOS

- 托盘在右上角菜单栏
- 默认不显示 Dock 图标（用户可配置）
- `NSAccessoryActivationPolicy` 控制

### Windows

- 系统托盘（右下角）
- 第一次右键可能需要点两次（这是 Windows 的 API 限制）

### Linux

- 依赖 `StatusNotifierItem` 协议
- KDE、GNOME 表现略有差异
- 某些 WM 需要额外安装 `libappindicator`
